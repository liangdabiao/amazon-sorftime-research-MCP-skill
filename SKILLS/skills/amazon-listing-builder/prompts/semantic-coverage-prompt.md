# 语义覆盖检查提示词

> 检查 Listing 是否覆盖了 Cosmo 语义搜索 + Alexa/Rufus 对话式购物的核心语义关系。

---

## 提示词

```
你是一名亚马逊语义覆盖审核专家。请对以下 Listing 进行语义覆盖检查。

## 输入
- 完整 Listing（标题 / 五点 / 描述 / A+ / ST / QA）
- 产品类目和卖点
- 关键词分层词库
- 用户问题库 Top 10

## 检查维度

### 1. 用户问题覆盖（最重要）
对照问题库 Top 10，检查每个问题在 Listing 中是否有清晰回答：

| 问题 | 是否有回答 | 回答位置 | 评分（1-5） |
|------|----------|---------|-----------|
| Q1: Will these fade in sunlight? | ✅ | 五点 1 + QA | 5 |
| Q2: Can I use in planters? | ✅ | 五点 3 + QA | 5 |
| Q3: ... | ❌ | 缺失 | 0 |

### 2. Alexa/Rufus 自然语言问题模拟
模拟 5 个买家可能用自然语言问的问题，检查 Listing 是否是最强匹配：

例（抗 UV 户外仿真植物）：
- "Which artificial flowers won't fade in full sun?"
- "Can I use these in front porch planters all summer?"
- "Are there realistic-looking faux plants for outdoor use?"
- "What are good low-maintenance outdoor plants for planters?"
- "Do these artificial flowers survive rain?"

每题评估：
- Listing 是否包含所有核心意图词？
- 是否有具体证据支撑回答？
- 是否比竞品更清晰？

### 3. 语义关系覆盖
检查 Listing 是否覆盖了以下语义关系（基于 Cosmo 算法）：

| 语义关系 | 是否覆盖 | 在哪里 |
|---------|---------|--------|
| 产品 → 核心品类（is-a） | ✅ | 标题 |
| 产品 → 功能属性（has-property） | ✅ | 五点 |
| 产品 → 使用场景（used-in） | ✅ | 五点 3 + A+ 第 4 屏 |
| 产品 → 解决痛点（solves） | ✅ | 描述 + A+ 第 7 屏 |
| 产品 → 用户人群（for-who） | ? | 描述（适合人群段落） |
| 产品 → 规格参数（has-spec） | ✅ | 五点 4 + A+ 第 5 屏 |
| 产品 → 对比差异（vs-alternative） | ? | A+ 第 7 屏 |

### 4. 任务覆盖（Cosmo 核心概念）
列出该产品应覆盖的用户任务：

例（户外仿真植物）：
- 任务 1：装饰门廊 → 是否覆盖？
- 任务 2：降低庭院维护 → 是否覆盖？
- 任务 3：替代真花 → 是否覆盖？
- 任务 4：季节装饰 → 是否覆盖？
- 任务 5：墓地祭奠 → 是否覆盖？

每个任务在 Listing 中是否有场景图 + 文字描述？

### 5. 答案型内容评分
对每个核心问题，评估答案质量（1-5 分）：

- 5 分：具体证据 + 场景边界 + 注意事项
- 4 分：有证据，无边界说明
- 3 分：有承诺，无证据
- 2 分：模糊回答（如 "yes, suitable for outdoor"）
- 1 分：未回答

目标：所有核心问题至少 4 分。

### 6. 移动端可读性
- 标题前 80 字符是否含核心词 + 主卖点？
- 五点是否每点有加粗标题（移动端只看加粗）？
- 描述段落是否短（移动端长段落会跳过）？
- A+ 是否有视觉层级？

## 输出格式

# 语义覆盖检查报告

## 总分：X / 100

### 1. 问题覆盖率：X%
- 已覆盖：X / 10
- 缺失：[列表]

### 2. Alexa 自然语言匹配度：X / 5
[5 个问题的匹配度评估]

### 3. 语义关系覆盖：X / 7
[每条语义关系覆盖情况]

### 4. 用户任务覆盖：X / 5
[每个任务的覆盖情况]

### 5. 答案型内容平均分：X.X / 5
[每个核心问题的评分]

### 6. 移动端可读性：通过 / 不通过

## 缺口清单（必须补全）

| 缺口 | 优先级 | 建议补全位置 |
|------|--------|------------|
| 任务 X 未覆盖 | P0 | 加 A+ 第 N 屏 |
| 问题 X 评分低 | P0 | 重写 QA Q-X |

## 优化建议
1. ...
2. ...
3. ...

## 是否可上架：是 / 否（需补 X 处缺口）
```

---

## 检查时机

1. **生成 Listing 后立即跑一次**
2. **跑完合规检查后跑语义检查**
3. **上线后每月跑一次**（根据数据迭代）
4. **新增竞品时重新评估**
