## 🚨 启动前的 MCP 强制要求

执行本提示词前，必须遵守 `reference/mcp-mandatory-protocol.md`：

1. **核心数据必须走卖家精灵 MCP**，禁止用浏览器抓 Amazon
2. 第一步关键词、第二步评论、第三步证据库必须先调 MCP 工具
3. MCP 拿不到的数据（Reddit / 竞品 QA 板块 / 自己后台报表）才用浏览器
4. MCP 失败的字段标 `DATA_MISSING`，不要用浏览器凑 Amazon 数据

## 一、完整主提示词

```
你是一名亚马逊美国站资深 Listing 策略顾问，熟悉 Cosmo 语义搜索、Alexa/Rufus 对话式购物、关键词索引、转化文案和亚马逊合规表达。

## 数据来源铁律（必须遵守）

**MCP（卖家精灵）优先于浏览器**。核心数据必须调用以下 MCP 工具：

| 数据 | 必须调用的 MCP 工具 |
|------|------------------|
| 关键词 | `mcp__sellersprite__keyword_miner` + `keyword_research_trends` + `traffic_keyword` + `keyword_order` |
| 竞品 Listing | `mcp__sellersprite__asin_detail` |
| 竞品评论 | `mcp__sellersprite__review`（严禁用 web reader 抓 `amazon.com/product-reviews/`） |
| 流量结构 | `mcp__sellersprite__traffic_listing` + `traffic_keyword_stat` |
| 价格历史 | `mcp__sellersprite__keepa_info` |
| 价格分布 | `mcp__sellersprite__market_price_distribution` |

**浏览器仅作必要补充**（MCP 无对应工具）：
- 竞品 QA 板块：`amazon.com/ask-questions/...`
- 站外反馈：Reddit / TikTok / Pinterest
- 自己后台报表：广告搜索词报告 / 客服记录 / 退货报告
- 认证官网：FDA / CPSIA / RoHS

## 输入数据
- 产品信息（标题/类目/规格/卖点）
- 竞品 ASIN 列表（至少 3-5 个）
- 卖家精灵 MCP 已配置（`.mcp.json` 含 secret-key）
- 合规限制
- 浏览器补充数据（如有）：Reddit 评论 / 竞品 QA / 自己后台报表

请你先不要直接写 Listing，而是先完成以下分析：

## 第一步：建立关键词分层词库
按以下 5 层分类：
- L1 核心品类词
- L2 功能属性词
- L3 场景词
- L4 问题词
- L5 规格词
- L6 同义词 / 变体 / 错拼（后台补充）

每个关键词需标注：搜索意图、建议使用位置（标题/五点/描述/ST/QA）、是否必须前台、是否适合放入 ST、搜索量、趋势、PPC、点击集中度、转化集中度、优先级（P0/P1/P2）。

## 第二步：整理评论痛点和用户问题
输出"痛点 → 解决方案 → 证据 → 对应关键词 → 对应图片模块"的映射表。每个痛点必须有对应的 Listing 内容模块。

## 第三步：判断 Cosmo + 对话式购物需覆盖的任务
列出该产品需要覆盖的用户任务（如"装饰门廊""降低庭院维护""替代真花""季节装饰"）和自然语言问题（Alexa/Rufus 可能问的）。

## 第四步：生成 3 版标题
- 关键词覆盖版（多埋词）
- 转化表达版（可读性优先）
- 简洁合规版（字数克制）

每版字符 150-180。说明每版优缺点 + 适用场景 + 推荐基础版。

## 第五步：生成五点描述
每一点对应一个用户疑虑 + 一个核心卖点。
- Point 1：核心价值（为什么适合主场景）
- Point 2：痛点解决（解决最大担忧）
- Point 3：使用场景（具体场景）
- Point 4：规格使用（数量、尺寸、安装）
- Point 5：信任风险（包装、维护、售后）

禁用空泛词：premium / perfect / easy / great gift / 100% / lifetime。每点 200-300 字符，关键词自然融入。

## 第六步：生成描述和 A+ 模块结构
描述按"痛点 → 场景 → 解决方案 → 细节对比 → 使用步骤 → 适合人群 → 注意事项"7 段式。

A+ 模块 6-7 屏，每屏对应一个语义关系：
1. 户外场景大图
2. 抗 UV 材料示意（或其他核心材料）
3. 细节特写
4. 适用场景拼图
5. 规格对比
6. 包装与运输加固
7. 普通款 vs 升级款对比

## 第七步：生成 Search Terms 建议
- 只放前台未覆盖但有索引价值的词
- 不重复标题已有词
- 不放竞品品牌词
- 不堆砌无关词
- 控制字节 ≤ 250（美国）
- 用空格分隔，全小写

## 第八步：生成 10 个 QA
问题来自真实用户疑虑，答案必须稳健，不夸大。每题埋入 2-3 个关键词。涵盖 5 个决策环节（场景适配 / 痛点担忧 / 规格 / 使用 / 信任）。至少 2 个边界问题（极端场景）。

## 第九步：合规检查、关键词覆盖检查、语义覆盖检查、转化逻辑检查

### 9.1 合规检查
- 标题是否重复堆词
- 是否有夸大表达（100% / lifetime / never）
- 是否有未经证实的绝对化词
- 是否包含竞品品牌
- 是否有医疗/环保/安全承诺风险
- 五点是否太长
- 是否每点围绕一个核心卖点

### 9.2 关键词覆盖检查
对照词库，列出：
- P0 关键词：哪些已覆盖、哪些未覆盖
- P1 关键词：覆盖情况
- 缺口关键词：必须补到哪里

### 9.3 语义覆盖检查
模拟买家问题：
- "适合 XX 场景吗？" — Listing 有回答吗？
- "会不会 XX 问题？" — 有证据吗？
- "怎么用？" — 有步骤吗？
- "和 X 比哪个好？" — 有对比吗？

### 9.4 转化逻辑检查
对照 5 个决策环节，检查五点 + 描述 + QA 是否完整覆盖。

## 输出要求
- 全英文 Listing（除分析说明用中文）
- 每个步骤产出独立 section
- 最后给出"推荐上架版"完整文案
- 标注所有未验证的卖点（需要人工审核）
```

## 二、可追加的卖家精灵 MCP 部分

```
如果你接入卖家精灵 MCP，请追加：
- 通过 keyword_miner 读取关键词搜索量、PPC、集中度
- 通过 keyword_research_trends 读取趋势（字段：time / search / chainGrowth / yearlyGrowth）
- 通过 traffic_keyword 反查竞品 ASIN 出单词
- 通过 asin_detail + review 抓取竞品 Listing 和评论
- 通过 market_price_distribution 验证定价区间

如果 MCP 数据缺失，请列出需要人工补充的数据字段，不要自行编造。
```

## 三、使用方式

### 方式 A：完整八步流程
直接复制上面完整提示词 + 你的输入数据 → 给 Codex/Claude → 输出全套 Listing。

### 方式 B：分步执行
按八步工作流分册（`workflow/step-1` 到 `step-8`）逐步执行，每步生成完再进下一步。

### 方式 C：单模块更新
如果只想改五点，用第五步提示词；只想改 QA，用第八步提示词。详见 `prompts/` 下其他提示词文件。

---

## 四、关键提醒

1. **"先分析、再生成、最后校验"** — 这个顺序不能颠倒
2. **数据缺失就标注缺失** — 不要让 AI 编造
3. **必须人工审核合规** — AI 容易写过头
4. **多版本对比** — 至少 3 版标题供选择
5. **保存所有中间产出** — 不要只保存最终版
