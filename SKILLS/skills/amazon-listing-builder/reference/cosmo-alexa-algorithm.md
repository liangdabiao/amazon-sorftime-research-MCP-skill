# COSMO 算法 + Alexa / Rufus 对话式购物：卖家应该怎么理解

> 这是 skill 的核心理论背景。理解清楚后，才知道为什么 Listing 要按这套方法做。

---

## 一、COSMO 是什么

COSMO 是 Amazon Science 公开发表的**电商常识知识生成与服务系统**。

**简单理解**：平台正在把"商品、场景、属性、用户需求、常识关系"连接起来。

### 给卖家的 3 个启发

1. **平台会理解商品常识**，不只是商品文本
   - 户外仿真植物不只是 decor，它还和 garden、patio、porch、planter、UV、fade resistant、low maintenance 这些概念有关

2. **平台会尝试理解用户购物任务**
   - 用户不是在找一个产品名，而是在完成一个任务
   - 装饰门廊、降低庭院维护、替代真花、墓地祭奠、季节装饰

3. **平台会把商品和需求的关系做得更细**
   - 你说自己适合户外，不够
   - 你要证明为什么适合户外：材料、抗 UV、颜色稳定、抗水、枝干强度、使用场景、客户反馈

## 二、Alexa / Rufus 对话式购物趋势

**重要边界**："Alexa 算法"**不是卖家后台明确公布的 Listing 评分规则**。

更准确地说，它代表的是：**Alexa、Rufus、对话式 AI 购物助手带来的自然语言购物趋势**。

### 趋势表现

| 过去 | 现在 / 未来 |
|------|-----------|
| 搜 "outdoor artificial flowers" | 问 "Which artificial flowers won't fade in full sun?" |
| 搜 "fake plants outdoor" | 问 "Can I use these in front porch planters all summer?" |
| 短关键词 | 完整自然语言问题 |

### 对 Listing 的影响

Listing 要从"关键词覆盖"升级到"语义覆盖"：
- 用户问题 → Listing 必须有答案
- 自然语言搜索 → QA 板块权重提升
- 多场景适配 → A+ 模块化叙事
- 真实证据 → 卖点必须有事实支撑

---

## 三、新方法 vs 旧方法

| 维度 | 旧方法 | 新方法 |
|------|--------|--------|
| 词的处理 | 堆砌覆盖 | 分层组织 |
| 标题 | 多埋词 | 结构化（品牌+核心+差异+场景+规格） |
| 五点 | 卖点罗列 | 决策链（5 个决策环节） |
| 描述 | 重复五点 | 叙事流（7 段式） |
| ST | 垃圾桶 | 补充索引池 |
| QA | 可有可无 | 答案型补丁（对话式搜索） |
| A+ | 产品特写 | 场景语义模块 |
| 关键词 | 是否放进去 | 关系有没有讲清楚 |

---

## 四、根本逻辑变化

### 旧逻辑
> 用户搜词 → 系统匹配词 → 商品展示 → 买家下单

### 新逻辑
> 用户在 TikTok 看场景视频 → 产生需求 → 在亚马逊搜短词 / 直接问 Alexa → 系统理解语义关系 → 推荐最强匹配 → 买家看图、看评论、看 QA → 决策

### 关键差异

**买家买的不是"产品名词"，而是"解决方案"**。

例：买家买的不是"artificial flowers"，而是"让户外空间更好看，同时不用维护，还不能很假"。

---

## 五、COSMO + Alexa 给 Listing 的 5 个新要求

### 要求 1：从"词覆盖"到"语义覆盖"
- 不只是把词埋进去，而是把"产品-场景-属性-问题-证据"的关系讲清楚
- 检查方法：每个核心关键词背后，Listing 是否有对应的语义模块

### 要求 2：从"产品名"到"用户任务"
- 不只是说"我卖什么"，而是"我解决什么任务"
- 检查方法：列出 5 个用户任务，Listing 是否都覆盖了

### 要求 3：从"卖点声明"到"证据支撑"
- 不只说"UV resistant"，要证明"为什么 UV resistant"
- 检查方法：每个卖点是否有材料/工艺/数据/认证证据

### 要求 4：从"短词"到"自然语言问题"
- 不只埋短关键词，要回答用户的完整问题
- 检查方法：模拟 5 个 Alexa 问题，Listing 是否是最强匹配

### 要求 5：从"一次性写"到"数据迭代"
- Listing 不是写完就完，要根据广告、QA、评论数据持续优化
- 检查方法：每月跑一次数据迭代分析（参考 `prompts/data-iteration-prompt.md`）

## 六、不神化算法，但要重视方向

### 不要做的
- ❌ 找"隐藏规则"和"神秘写法"
- ❌ 把 Cosmo 讲成玄学
- ❌ 认为关键词不重要了

### 要做的
- ✅ 让 Listing 更相关（精准匹配用户任务）
- ✅ 让 Listing 更清楚（结构化、可视化）
- ✅ 让 Listing 更可信（每个卖点都有证据）
- ✅ 让 Listing 更能转化（按决策链写五点）
- ✅ 让 Listing 能持续迭代（数据驱动）

---

## 七、一句话总结

> **算法怎么变，最终都围绕一个问题：给用户展示最相关、最有可能成交、体验最好的商品。**

所以你要做的不是迎合玄学，而是让 Listing：
1. **更相关** — 关键词分层，场景精准
2. **更清楚** — 结构化、可视化、移动端友好
3. **更可信** — 每个卖点都有证据
4. **更能转化** — 按决策链写五点，QA 补疑虑
5. **可迭代** — 数据驱动持续优化
