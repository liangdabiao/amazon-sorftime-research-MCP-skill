# 输出格式规范

> 所有 Listing 产出必须保存为文件，禁止只打印到控制台。

---

## 一、目录结构

```
{产品名}/
├── listing_package.md           # 完整 Listing 包（汇总）
├── keyword_library.md           # 关键词分层词库（第一步）
├── question_library.md          # 用户问题库（第二步）
├── evidence_library.md          # 卖点证据库（第三步）
├── drafts_comparison.md         # 多版本草稿对比（第四五六步）
├── audit_report.md              # 合规 + 语义 + 转化校验（最终）
└── input_data.json              # 原始输入数据
```

---

## 二、文件命名规则

| 文件 | 命名 | 必存 |
|------|------|:----:|
| 完整 Listing 包 | `{产品英文小写_连字符}/listing_package.md` | ✅ |
| 关键词词库 | `keyword_library.md` | ✅ |
| 问题库 | `question_library.md` | ✅ |
| 证据库 | `evidence_library.md` | ✅ |
| 草稿对比 | `drafts_comparison.md` | ✅ |
| 校验报告 | `audit_report.md` | ✅ |
| 原始数据 | `input_data.json` | ✅ |
| HTML 可视化（可选） | `listing_package.html` | ⭐ 推荐生成 |

### 目录命名示例
- `uv-outdoor-plants/`（抗 UV 户外仿真植物）
- `down-filled-pillows/`（羽绒枕）
- `wireless-instruments/`（无线麦克风）

## 六、HTML 可视化报告（推荐生成）

完成 `listing_package.md` 后，**推荐自动生成** `listing_package.html`：

### 设计要求
1. 深色专业主题、配色统一
2. 响应式适配
3. 关键数据用图表（关键词分布饼图、覆盖率柱状图）
4. 单文件 HTML + CSS + JS（无外部依赖）

### 内容覆盖
1. **概览** — KPI 摘要（关键词数、覆盖率、合规分数）
2. **关键词分布** — 5 层词库可视化
3. **痛点-证据映射** — 关系图
4. **标题对比** — 3 版对比卡片
5. **五点 / 描述 / QA** — 卡片式展示
6. **校验报告** — 雷达图（合规 / 覆盖 / 语义 / 转化）

> 用户明确说"不需要 HTML"可跳过。

## 七、报告质量检查清单

| # | 检查项 |
|---|--------|
| 1 | 所有文件已保存到 `{产品目录}/` 下 |
| 2 | 关键词无 HTML 实体（`&amp;` 等） |
| 3 | 数据缺失字段标 DATA_MISSING |
| 4 | 数值格式化（销量千分位、价格 $X.XX） |
| 5 | 表格对齐（`:---` `:---:` `---:`） |
| 6 | 标题层级 ≤ 4 级 |
| 7 | 筛查口径已记录（用了哪些工具、什么时候的数据） |
| 8 | 异常数据有 ⚠️ 标注 |

## 八、数值格式化规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 销量 | 千分位 | 12,345 |
| 销量占比 | 1 位小数 | 23.4% |
| 评分 | 1 位小数 | 4.2 |
| 价格 | $ + 2 位小数 | $24.99 |
| 集中度（0-1） | ×100 + 1 位小数 | 42.5% |
| 转化率 | 1 位小数 | 6.5% |
| CPC | $ + 2 位小数 | $0.85 |
| 趋势百分比 | + 号 + 1 位小数 | +12.5% |

---

## 九、关键铁律

1. **禁止只打印到控制台** — 所有产出必须保存为文件
2. **MD 报告完成后推荐生成 HTML** — 提升可读性
3. **每个文件命名一致** — 用 `keyword_library.md` 而非 `keywords.md`
4. **保留所有中间产出** — 不要只保存最终版
5. **校验报告必须诚实** — 不能为了让报告好看而隐藏问题
